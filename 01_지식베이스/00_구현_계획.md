# ARWay Lite 구현 계획

## 프로젝트 개요

**ARWay Lite**는 SCQ 기반 AR 도보 네비게이션 MVP로, 로컬 웹 환경에서 PoC를 수행합니다.

## 전체 워크플로우 순서

### Phase 1: 인프라 및 백엔드 (1-3주)

1. **1.00 프로젝트 초기 설정** (1일)
   - 프로젝트 구조 생성
   - 개발 환경 설정
   - 필수 도구 설치

2. **2.00 데이터베이스 설계 및 마이그레이션** (2일)
   - ERD 기반 모델 정의
   - Alembic 마이그레이션 생성
   - 시드 데이터 생성

3. **3.00 백엔드 API 서버 구축** (3일)
   - FastAPI 앱 초기화
   - API 라우터 구현
   - CRUD 엔드포인트 완성

### Phase 2: 프론트엔드 사용자 화면 (2-3주)

4. **4.00 프론트엔드 시작 화면 구현** (1일)
   - 시작 화면 레이아웃
   - 라우팅 설정

5. **5.00 목적지 선택 화면 구현** (2일)
   - 목적지 목록 표시
   - 선택 기능 구현
   - 세션 생성

6. **6.00 AR 네비 실행 화면 구현** (5일) ⭐ 핵심
   - 카메라 프리뷰
   - GPS 추적
   - 방향 화살표
   - 거리 계산

7. **7.00 도착 화면 구현** (1일)
   - 도착 메시지
   - 피드백 수집

### Phase 3: 관리자 및 완성 (1-2주)

8. **8.00 Admin 관리자 화면 구현** (3일)
   - 대시보드
   - 세션 관리
   - 목적지 관리

9. **9.00 통합 테스트 및 배포** (2일)
   - 테스트 작성
   - 통합 테스트
   - 배포 스크립트

## 기술 스택 요약

- Next.js 14+ (App Router)
- TypeScript
- TailwindCSS
- geolib
- Zustand

### Backend
- FastAPI
- SQLAlchemy
- Alembic
- PostgreSQL

### Infrastructure
- Docker Compose
- PostgreSQL 15+

## 주요 마일스톤

### Milestone 1: 백엔드 완성 (1주차 말)
- 데이터베이스 스키마 완성
- 모든 API 엔드포인트 구현 완료
- API 문서 생성

### Milestone 2: 핵심 기능 완성 (2주차 말)
- AR 네비게이션 화면 구현 완료
- GPS 추적 정상 동작
- 방향 화살표 정상 동작

### Milestone 3: PoC 완성 (4주차 말)
- 모든 화면 구현 완료
- 통합 테스트 완료
- 로컬 배포 완료

## 리스크 관리

### 기술적 리스크
1. **GPS 정확도 문제**
   - 대응: 고정밀 GPS 설정, 평균값 사용

2. **디바이스 헤딩 지원 문제**
   - 대응: Fallback 로직 구현 (GPS 방향 계산)

3. **카메라 권한 문제**
   - 대응: 카메라 없이도 동작 가능하도록 처리

### 일정 리스크
1. **AR 네비 구현 지연**
   - 대응: 우선순위 조정, 핵심 기능 집중

2. **통합 테스트 시간 부족**
   - 대응: 단위 테스트 강화, 자동화

## 성공 기준

### 기능적 성공
- ✅ 모든 화면 정상 동작
- ✅ GPS 추적 정확도 ±10m 이내
- ✅ 방향 오차 ±15° 이내

### 기술적 성공
- ✅ 화면 FPS 30fps 이상
- ✅ API 응답 시간 <200ms
- ✅ 데이터베이스 쿼리 최적화

### 사용자 경험 성공
- ✅ 직관적인 UI/UX
- ✅ 부드러운 AR 화살표 회전
- ✅ 명확한 피드백 제공

## 다음 단계 (PoC 이후)

1. **사용자 피드백 수집**
   - 실제 사용자 테스트
   - 피드백 분석

2. **성능 최적화**
   - 렌더링 최적화
   - API 응답 시간 개선

3. **SCQ 모델 통합 준비**
   - 모델 인터페이스 설계
   - 통합 포인트 정의

4. **프로덕션 배포 계획**
   - 클라우드 인프라 설계
   - CI/CD 파이프라인 구축
